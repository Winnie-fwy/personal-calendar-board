# 手机配置与升级指南

## 先说明结论

当前版本已经是一个可部署的轻量 PWA 看板。它可以像 App 一样添加到安卓手机桌面，但它不是需要安装包的原生 App。

你手机上不能直接长期使用 `computer://` 或本地文件路径；正式使用时，需要把 `personal-calendar-board` 这个文件夹部署到一个 HTTPS 网站地址，例如 Vercel、Netlify、Cloudflare Pages 或 GitHub Pages。

部署后，你和对象都打开同一个网址，再添加到手机桌面。

## 版本更新会不会同步

### 如果使用 PWA 方式

大多数更新不需要重新下载。

只要你把新版代码部署到同一个网址，手机桌面图标打开的还是同一个 PWA 地址。下次打开时，页面会优先请求网络新版，同时保留离线缓存兜底。

通常表现是：

- 页面内容、样式、功能更新：重新打开或刷新后生效。
- 新增页面、新增 JS 逻辑：重新打开后生效。
- 图标、名称、缓存策略变化：可能需要多打开一次，或者从浏览器里刷新一次。
- 如果浏览器缓存很顽固：清除该网站数据，或删除桌面图标后重新添加。

### 如果以后做原生 App

原生 App 不一样。

如果是通过 APK 或应用商店安装，功能升级通常需要发新版本。用户需要安装更新包，或应用商店自动更新。

### 推荐策略

早期建议用 PWA。

原因是你会不断调整功能、字段、页面和四库结构，PWA 更适合快速迭代。等结构稳定后，再做 Flutter 原生 App 和桌面小组件。

## 第一步：部署到 HTTPS

推荐免费部署方式：

| 方式 | 推荐程度 | 说明 |
|---|---|---|
| Vercel | 高 | 简单，适合前端静态项目 |
| Netlify | 高 | 拖拽文件夹即可部署 |
| Cloudflare Pages | 高 | 免费额度友好，访问稳定 |
| GitHub Pages | 中 | 免费，但配置稍微麻烦 |

最简单方式是 Netlify：

1. 打开 Netlify。
2. 选择 Add new site。
3. 选择 Deploy manually。
4. 上传整个 `personal-calendar-board` 文件夹。
5. 得到一个 `https://...netlify.app` 地址。
6. 用手机浏览器打开这个地址。

## 第二步：OPPO 手机上添加到桌面

1. 用 OPPO 手机上的浏览器或 Chrome 打开部署后的网址。
2. 点击浏览器菜单。
3. 找到 `添加到桌面`、`添加到主屏幕` 或类似选项。
4. 名称建议填 `专属日历`。
5. 添加后，桌面会出现一个类似 App 的图标。
6. 以后直接点这个图标进入看板。

如果找不到添加到桌面：

- 换 Chrome 浏览器试一次。
- 确认网址是 HTTPS。
- 确认 `manifest.webmanifest` 和 `service-worker.js` 已部署成功。

## 第三步：小米手机添加到桌面

1. 用小米手机浏览器或 Chrome 打开部署后的网址。
2. 点击浏览器菜单。
3. 选择 `添加到桌面` 或 `添加到主屏幕`。
4. 允许创建桌面快捷方式。
5. 如果系统拦截，需要在权限设置里允许浏览器创建桌面快捷方式。

小米系统可能需要额外允许：

- 浏览器创建桌面快捷方式。
- 通知权限。
- 后台运行权限。

## 第四步：配置强提醒

当前 PWA 原型使用 `.ics` 文件导入系统日历提醒。

使用方式：

1. 打开看板。
2. 进入 `任务`。
3. 点击 `导入系统日历提醒`。
4. 手机会下载或打开一个 `今日强提醒示例.ics` 文件。
5. 用系统日历打开并导入。
6. 系统日历会按提醒时间通知你。

注意：

- PWA 本身的浏览器通知在安卓上不如系统日历稳定。
- OPPO 和小米对后台通知限制较多。
- 重要任务优先写入系统日历。
- 后续做独立 App 后，可以直接申请日历权限，把任务写入系统日历，不需要手动导入 `.ics`。

## 第五步：多人同步

当前版本的数据保存在手机本地。

这意味着：

- 你在自己手机新增的任务，当前不会自动出现在对象手机上。
- 对象手机也不会自动同步你的本地数据。
- 这是本地原型阶段的限制。

要实现真正多人同步，需要接入云数据库。

建议后续使用：

- Supabase 免费档。
- 邮箱登录。
- 双人空间。
- 四库数据表。
- 模块级权限。

## 后续升级流程

### PWA 升级流程

每次我帮你改完功能后：

1. 替换服务器上的旧文件。
2. 保持网址不变。
3. 打开手机桌面图标。
4. 如果没更新，关闭后重新打开。
5. 仍没更新，就用浏览器打开网址刷新一次。

本项目的 `service-worker.js` 已设置为网络优先策略，正常情况下会更容易拿到新版。

### 需要重新添加桌面的情况

以下情况可能需要删除桌面图标后重新添加：

- App 名称改了。
- 图标改了但手机一直不刷新。
- 网站域名换了。
- 从测试地址迁移到正式地址。
- 手机缓存异常。

### 不需要重新添加桌面的情况

以下情况通常不需要重新添加：

- 页面样式调整。
- 四库结构调整。
- 新增任务识别逻辑。
- 新增复盘逻辑。
- 新增知识库记录规则。
- 修复 bug。

## 下一步建议

现在最适合的路线是：

1. 继续把当前 PWA 做完整。
2. 部署到免费 HTTPS 地址。
3. 你和对象都添加到桌面。
4. 先用 1-2 周，验证四库结构和对话输入是否顺手。
5. 再接 Supabase 免费云数据库，实现双人同步。
6. 最后再做 Flutter 独立 App 和安卓桌面小组件。
