# 免费云数据库方案

## 结论

可以先不花钱做双人同步原型。你和对象两个人使用、以文本任务和知识库为主的场景，免费云数据库通常够用。

我更建议先用 `Supabase 免费档`，原因是它更像一个真正 App 后端：有数据库、账号登录、权限控制、文件存储和实时同步，后续迁移到独立 App 更自然。

## 推荐方案

### 首选：Supabase 免费档

适合你的原因：

- 可做账号系统。
- 可做你和对象的双人空间。
- 可做四个库：总日历库、任务库、知识库、复盘库。
- 可按模块设置共享权限。
- 可做实时同步。
- 数据结构接近正式 App 后端。

适合免费使用的内容：

- 今日计划。
- 任务打勾。
- 知识库文本笔记。
- 复盘总结。
- 双人共享权限。
- 少量头像或图片附件。

容易产生费用的内容：

- 大量图片、音频、视频附件。
- 长期保存大量日志。
- 很多人一起使用。
- 大量实时在线连接。
- 超出免费项目或存储限制。

### 备选：Firebase 免费档

适合你的原因：

- 安卓生态友好。
- 后续做 App 通知、登录、崩溃统计比较方便。
- 对 Flutter / React Native 支持成熟。

需要注意：

- Firestore 是 NoSQL 数据库，复杂关系和权限设计需要提前规划。
- 如果后续用量增长，计费方式更偏“按读写次数和存储用量”。

## 最省钱落地方式

第一版建议这样做：

| 模块 | 免费落地方式 |
|---|---|
| 账号 | 邮箱登录，暂时不要短信验证码 |
| 数据库 | Supabase 免费档 |
| 附件 | 少量图片，优先压缩 |
| 提醒 | 安卓系统日历，不依赖云端短信 |
| 同步 | 只同步任务、知识、复盘文本 |
| 权限 | 用共享表控制“谁能看什么” |

## 数据表建议

| 表 | 作用 |
|---|---|
| `users` | 你和对象的用户资料 |
| `spaces` | 双人共享空间 |
| `space_members` | 你和对象在空间里的角色 |
| `calendar_items` | 总日历库 |
| `tasks` | 任务库 |
| `knowledge_notes` | 知识库 |
| `reviews` | 复盘库 |
| `share_permissions` | 模块级共享权限 |
| `calendar_exports` | 系统日历导出记录 |

## 权限建议

| 内容 | 默认权限 |
|---|---|
| 今日完成率 | 对象可见 |
| 今日任务标题 | 对象可见 |
| 学习知识库 | 可共享 |
| 普通生活事项 | 可共享 |
| 健康细节 | 默认私密 |
| 投资金额 | 默认私密 |
| 投资提醒 | 可共享提醒，不共享金额 |
| 灵感收集 | 默认私密，手动分享 |

## 后续什么时候需要付费

以下情况才需要认真考虑付费：

- 你们长期上传大量图片、文件、录音。
- 想邀请更多人使用。
- 想要更稳定的生产环境和备份。
- 想做公开发布的独立 App。
- 数据库或文件存储超过免费额度。

如果只是你和对象两个人使用，第一版完全可以先从免费方案开始。
